简易的分布式锁

` setnx user:1001 true ` 是1时即返回成功，0则是获取失败 ### 分布式锁加强版 在上一个简易的分布式锁可以满足一定的需求，但是存在不少的问题。

  • 问题1

    当一个redis的用户端获取了锁以后，在处理业务的过成中，出现了突然停电或者其他突然导致线程死亡的情况，
    会使该客户端不能及时释放锁资源，导致该锁会永远处在上锁状态。

  • 解决

    在创建锁的同时给他加上有效期，超过有效期就删除。set user:1001 true ex 10 nx

  • 问题2

    该锁不支持可重入，同一线程获取两次锁将会造成死锁

  • 解决
    在set value 时设置uuid，创建锁时，返回锁的值，重入时判断锁的值是否相同

Java实现

/**
 * 获取分布式锁
 *
 * @param keyName
 * @param time    秒
 * @return
 */
public static boolean getLock(@NotNull String keyName, @NotNull String value, @NotNull Integer time) {
    //判断是否可以直接获取锁资源(可以重入)
    String lockValue = StringUtil.objectToString(redisTemplate.opsForValue().get(keyName));
    if (value.equals(lockValue)) {
        return true;
    }
    if (redisTemplate.opsForValue().setIfAbsent(keyName, value, Duration.ofSeconds(time))) {
        return true;
    }
    return false;
}

/**
 * 解锁
 */
public static void unLock(@NotNull String keyName, @NotNull String value) throws Exception {
    String lockValue = StringUtil.objectToString(redisTemplate.opsForValue().get(keyName));
    if (lockValue == null || lockValue.equals(value)) {
        redisTemplate.delete(keyName);
    }else {
        throw new Exception("无法解锁");
    }
}

  • 问题3

    如何合理的设置分布式锁的超时时间，如果在业务处理时间内分布式锁超时，该锁将会失去对资源的保护。

  • 解决

    在获取锁的主线程获取锁的同时生成子线程，子线程随主线程的消亡而停止(就是 守护线程的意思)。子线程的作用是，当锁的超时时间快到的时候重新为锁设置超时时间。

  • 问题4

    不想一个线程没有取到锁就之间放弃

  • 解决

    在尝试获取锁时增加尝试获取锁的次数或者增加获取时间

/**
 * 尝试获取锁
 * @param keyName
 * @param keyValue
 * @param time
 * @param tryTimes  尝试次数
 * @return
 */
public static boolean tryRedisLock(@NotNull String keyName, @NotNull String keyValue, @NotNull Integer time,
                                   Integer tryTimes) {
    if (tryTimes != null) {
        while (true) {
            boolean redisLock = getRedisLock(keyName, keyValue, time);
            if (redisLock) {
                return true;
            } else {
                tryTimes--;
                if (tryTimes == 0) {
                    return false;
                }
            }
        }
    } else {
        return getRedisLock(keyName, keyValue, time);
    }
}
