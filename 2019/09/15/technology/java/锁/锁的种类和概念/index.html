锁的种类和概念

本文介绍个人理解的大致的几个锁的概念

公平锁/非公平锁

  • 公平锁

    获取锁的顺序按照请求锁的时间的来排队

  • 非公平锁

    获取锁的顺序根据线程随机抢占资源

    可重入锁

    同一个线程获取锁资源以后在线程内部再次使用该锁时可以直接获取锁资源

独享锁/共享锁

独享锁一次只能被一个线程使用 - 读锁
共享锁一次能被多个线程使用   - 写锁

悲观锁/乐观锁

悲观锁是指先获取锁资源等到获取资源以后在进行业务上的操作
乐观锁是指先进行业务上的操作在去获取锁

自旋锁

当一个线程获取锁资源失败时不会直接退出，而是会多次尝试获取锁，当达到一定次数或者是其他设置的条件时才会退出

偏向锁

这三种锁是指锁的状态，并且是针对Synchronized
一把锁一直被一个线程所访问，成为偏向锁。
轻量级锁是指当锁是偏向锁的时候，被另一个线程所访问，偏向锁就会升级为轻量级锁，其他线程会通过自旋的形式尝试获取锁，不会阻塞，提高性能。
重量级锁是指当锁为轻量级锁的时候，另一个线程虽然是自旋，但自旋不会一直持续下去，当自旋一定次数的时候，还没有获取到锁，就会进入阻塞，该锁膨胀为重量级锁。重量级锁会让其他申请的线程进入阻塞，性能降低

分段锁

将整一个代码或者整一个业务根据合理的操作步骤进行分段，在每一段上加上锁。尽可能的减小代码的颗粒度，能够最大的提升锁的性能
