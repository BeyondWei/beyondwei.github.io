java集合

集合很重要

List/Set/Map 的区别

  • List 有序集合，多个元素截图可以引用相同的对象（可以出现两个相同的值）
  • Set 每个元素引用的对象不能重复（不能出现两个相同的值）
  • Map Key-Value方式存储数据，不可能出现相同的Key值

List

ArrayList和LinkedList

  • ArrayList 和 LinkedList 都不是同步的
  • ArrayList是数组，LinkedList
  • ArrayList对数据的查找比LinkedList快，LinkedList的插入删除操作比ArrayList快
  • LinkedList不支持高效的随机元素访问，而 ArrayList 支持。快速随机访问就是通过元素的序号快速获取元素对象(对应于get(int index)方法)

ArrayList和Vector

  • Vector 是同步的

Map

HashMap 和 Hashtable

  • HashMap是线程不安全的，Hashtable是线程安全的
  • Hashtable的线程安全机制是通过synchronized实现的，效率相对比较低，所以推荐使用ConcurrentHashMap
  • null 值可以在HashMap中做键值但是不能在Hashtable中做键值
  • HashMap的扩容机制：默认是11，每次扩容是2n+1，Hashtable的默认是16，每次扩容是2n
  • 1.8以后的版本HashMap 使用红黑树进行搜索

LinkedHashMap 和 TreeMap

  • LinkedHashMap 继承自 HashMap，所以它的底层仍然是基于拉链式散列结构即由数组和链表或红黑树组成。另外，LinkedHashMap 在上面结构的基础上，增加了一条双向链表，使得上面的结构可以保持键值对的插入顺序。同时通过对链表进行相应的操作，实现了访问顺序相关逻辑
  • TreeMap 红黑树（自平衡的排序二叉树）

HashMap 底层实现

  • JDK1.8 之前底层使用链表和数组（哈希桶）
  • JDK1.8 以后，当HashMap的链表长超过8时，链表转换为红黑树

ConcurrentHashMap 和 Hashtable

  • ConcurrentHashMap在JDK1.7时使用分段数组+链表，在JDK1.8时使用数组+链表/红黑树；Hashtable使用数组+链表的方式
  • 在JDK1.7时ConcurrentHashMap 对Segment 使用分段锁，在JDK1.8时使用Node数组+链表/红黑树，synchronized 和CAS，Hashtable使用全局锁实现同步

Set

HashMap 和 HashSet

  • 基于 HashMap 实现的，底层采用 HashMap 来保存元素
  • HashMap是通过Key来计算HashCode，HashSet是通过对象来计算HashCode
  • HashSet 先计算HashCode是否有重复，如果没有重复就会直接放入Set中，如果出现HashCode重复就会通过equals方法判断，如果equals也相同就不会加入集合

LinkedHashSet 和 TreeSet

  • LinkedHashSet 继承于 HashSet，并且其内部是通过 LinkedHashMap 来实现的。有点类似于我们之前说的LinkedHashMap 其内部是基于 HashMap 实现一样，不过还是有一点点区别的
  • TreeSet 红黑树(自平衡的排序二叉树)

集合的选择

  • 主要根据集合的特点来选用，比如我们需要根据键值获取到元素值时就选用Map接口下的集合，需要排序时选择TreeMap,不需要排序时就选择HashMap,需要保证线程安全就选用ConcurrentHashMap.当我们只需要存放元素值时，就选择实现Collection接口的集合，需要保证元素唯一时选择实现Set接口的集合比如TreeSet或HashSet，不需要就选择实现List接口的比如ArrayList或LinkedList，然后再根据实现这些接口的集合的特点来选用。
