redis配置详细详解（6.AOF配置）

人在宁波，心在杭州

appendonly

  • 模板：appendonly no
  • 作用：是否开启AOF持久化存储

appendfilename

  • 模板：appendfilename “appendonly.aof”
  • 作用：aof存储文件名称

appendfsync

  • 模板：appendfsync everysec

  • 作用：数据存储到AOF中的策略

  • 参数

    参数 说明
    always 每一次操作都更新到AOF中，这样的数据时最安全的，但是需要占用较大的数据
    no redis 不会主动的更新到AOF中，而是由操作一同刷新磁盘的时候写入到AOF中
    everysec 每秒更新一次到AOF，是安全和性能的折中选择

no-appendfsync-on-rewrite

  • 模板：no-appendfsync-on-rewrite no
  • 作用：在子进程重写AOF和主线程写入磁盘同时进行时解决冲突方法。如果应用系统无法忍受延迟，而可以容忍少量的数据丢失，则设置为yes。如果应用系统无法忍受数据丢失，则设置为no

auto-aof-rewrite-percentage

  • 模板：auto-aof-rewrite-percentage 100
  • 作用：在redis写入AOF的过程中产生的操作日志会暂时存在在缓存中，当前缓存的大小比上一次写的缓存的大小到指定的百分比的时候才会再次追加到AOF中

auto-aof-rewrite-min-size

  • 模板：auto-aof-rewrite-min-size 64mb
  • 作用：缓存至少要达到指定的值才能被自动追加到AOF文件中

lua-time-limit

  • 模板：lua-time-limit 5000
  • 作用：一个 Lua 脚本最长的执行时间为 5000 毫秒（ 5 秒），如果为 0 或负数表示无限执行时间。
