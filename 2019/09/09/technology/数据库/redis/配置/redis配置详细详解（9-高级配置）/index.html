redis配置详细详解（9.高级配置）

Redis 配置文件的最后一篇

hash-max-zipmap-entries hash-max-zipmap-value

  • 模板：hash-max-ziplist-entries 512；hash-max-ziplist-value 64
  • 作用：当hash 中包含超过指定元素个数并且最大的元素没有超过临界时，hash 将以一种特殊的编码方式（大大减少内存使用）来存储，这里可以设置这两个临界值

list-max-ziplist-entries list-max-ziplist-value

  • 模板：list-max-ziplist-entries 512；list-max-ziplist-value 64
  • 作用：当list 中包含超过指定元素个数并且最大的元素没有超过临界时，list 将以一种特殊的编码方式（大大减少内存使用）来存储，这里可以设置这两个临界值

set-max-intset-entries

  • 模板：set-max-intset-entries 512
  • 作用：set 数据类型内部数据如果全部是数值型，且包含多少节点以下会采用紧凑格式存储

zset-max-ziplist-entries zset-max-ziplist-value

  • 模板：zset-max-ziplist-entries 128；zset-max-ziplist-value 64
  • 作用：当zset 中包含超过指定元素个数并且最大的元素没有超过临界时，zset 将以一种特殊的编码方式（大大减少内存使用）来存储，这里可以设置这两个临界值

hll-sparse-max-bytes

  • 模板：hll-sparse-max-bytes 3000
  • 作用：HyperLogLog优化的最大字节

activerehashing

  • 模板：activerehashing yes
  • 作用：Redis 将在每 100 毫秒时使用 1 毫秒的 CPU 时间来对 redis 的 hash 表进行重新 hash ，可以降低内存的使用。 当你的使用场景中，有非常严格的实时性需要，不能够接受 Redis 时不时的对请求有 2 毫秒的延迟的话，把这项配置为 no。 如果没有这么严格的实时性要求，可以设置为 yes ，以便能够尽可能快的释放内存

client-output-buffer-limit

  • 模板：client-output-buffer-limit normal 0 0 0 硬限制，软限制，时间
  • 作用：客户端的输出缓冲区的限制，因为某种原因客户端从服务器读取数据的速度不够快,一旦达到硬限制客户端会立即断开，或者达到软限制并保持达成的指定秒数（连续）,把硬限制和软限制都设置为 0 来禁用该特性
  • 参数
    参数 说明
    normal 正常客户端
    slave slave 和 MONITOR 客户端
    pubsub 至少订阅了一个 pubsub channel 或 pattern 的客户端

hz

  • 模板：hz 10
  • 作用： Redis 调用内部函数来执行许多后台任务，如关闭客户端超时的连接，清除过期的 Key ，等等。Redis 依照指定的“ Hz ”值来执行检查任务。范围是 1 到 500 之间，但是值超过 100 通常不是一个好主意

aof-rewrite-incremental-fsync

  • 模板：aof-rewrite-incremental-fsync yes
  • 作用：当一个子节点重写 AOF 文件时，如果启用下面的选项，则文件每生成 32M 数据进行同步。
