redis配置详细详解（1.基本配置信息）

博客每日一更，今日打卡

使用配置文件文件须知

  • 启动redis时需要指定配置文件地址，否则会使用默认配置
  • 可以同时使用多个配置文件，但是当配置文件同有设置内容的话，会以最后一个配置为准

基本配置信息

daemonize

  • 模板：daemonize yes
  • 作用：是否开启守护进程，守护进程模式运行相当于后台运行

pidfile

  • 模板：pidfile /var/run/redis/redis-server.pid
  • 作用：在开启守护进程的模式的情况下，会生成一个记录redis运行端口的文件

port

  • 模板：port 3679
  • 作用：redis运行使用的端口号

tcp-backlog

  • 模板：tcp-backlog 511
  • 作用：完成tcp三次连接以后等待处理的队列长度

bind

  • 模板：bind 127.0.0.1
  • 作用：允许连接的地址，注释掉就可以开放所有地址

timeout

  • 模板：timeout 0
  • 作用：空闲时间超过指定时间将会断开连接，0表示禁用

tcp-keepalive

  • 模板：tcp-keepalive 0
  • 作用：在不为0的情况下，会通过指定时间的一半去向客户端发送ack信息，来询问连接是否存活

loglevel

  • 模板：loglevel notice

  • 作用：日志记录级别

  • 参数说明

    参数名 说明
    debug 很多信息，对开发/测试有用
    verbose 许多很少有用的信息，但不像调试级别那样混乱
    notice 可能有些冗长，您在生产中需要什么
    warning 只记录非常重要/关键的消息

logfile

  • 模板：logfile /var/log/redis/redis-server.log
  • 作用：日志记录地址

databases

  • 模板：databases 16
  • 作用：redis数据库创建的数量
